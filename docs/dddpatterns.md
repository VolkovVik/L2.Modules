# **Тактические паттерны DDD**

## _Value Object_

Value Object — это объект, который можно идентифицировать по составу его значений.

### Признаки
- Нет выделенного идентификатора
- Идентифицировать объект можно по составу его значений
- Обычно является свойством другого объекта

### Хранение в БД
- Связь «One-to-one» с материнским объектом
- Не имеет отдельной таблицы. Храниться как под колонки в таблице материнского объекта

## _Entity_

Entity (сущность) - представляет концепцию в вашей предметной 
области, которая определяется ее идентичностью, а не ее 
атрибутами.

### Признаки
- Доменный объект требует явного идентификационного поля, чтобы различать разные экземпляры объекта
- Не самостоятельная сущность, а часть материнской (агрегата)

### Хранение в БД
- Связь «One-to-many» или «Many-to-one» с материнским объектом
- Храниться в отдельной таблице

## _Aggregate_

Aggregate (агрегат) - это кластер объектов предметной области, 
которые можно рассматривать как единое целое. Примером может 
быть заказ и его позиции.
Aggregate (агрегат) является границей обеспечения 
согласованности. Логика агрегата должна проверять все входящие 
изменения и гарантировать, что изменения не противоречат его 
бизнес-правилам.

### Признаки
- Самостоятельный объект
- Требует явного идентификатора
- Является материнским по отношению к другим объектам

### Хранение в БД
- Связь «One-to-one» с Value Object
- Связь «One-to-many» или «Many-to-one» с Entity
- Хранится в отдельной таблице
